# CastLock-Vis 中期报告

> 注：本文是中期报告归档，保留了当时（2026-06-01）的阶段叙述与截图说明。项目已在 S5 后切换到专家流水线数据与完整联动基线；当前权威状态请以 `README.md`、`docs/plan/TODO.md`、`docs/plan/FEATURE_LIST.md` 与 `docs/dev_rule/ARCHITECTURE.md` 为准。

## 1. 项目背景

在影视行业里，演员往往会被市场和制片体系“类型化”——长期困在动作、喜剧、剧情等少数赛道中，难以突破。我们关注的是这种“类型锁定（cast lock）”现象背后的两个核心问题：演员在职业生涯中后期，是否真的会被逐步焊死在单一类型里？以及，如果存在转型的可能，那么转型的时机和条件分别是什么？

传统的统计表格虽然能给出均值和比例，但很难展现“从群体到个体”的演化过程，也难以捕捉那些关键转折点。为了弥补这一空白，我们构建了可视分析系统 CastLock-Vis，通过多视图联动的方式，实现从宏观群落识别到微观转型轨迹审计的递进式分析。

## 2. 分析任务定义

围绕"类型锁定与转型窗口期"这一主题，我们定义了四个层次的分析任务。

在宏观层面，首先是**类型群落与基线识别**。我们需要识别出演员在早期职业生涯中形成的天然群落结构——比如纯粹的动作片群落、喜剧群落或剧情片群落——以此建立“舒适圈”的基线认知。

进入中观层面，第二个任务是**多元度衰减与窗口期探测**。我们利用类型香农熵曲线来观察演员生涯序列中类型多元度的变化趋势，从而定位某个类型或某位演员的潜在转型窗口期。

第三个任务是**类型转移机制审计**。我们通过分阶段的马尔可夫转移矩阵，展示演员主演的电影从“当前类型”跳向“下一部类型”的变化概率，分析其选片路径是否存在收缩现象，并检验类型锁定的"重力场"是否会随着演员生涯推进而增强。

在微观层面，第四个任务是**转型分叉与生存分析**。我们以演员的首次转型事件为 T=0 进行对齐，比较成功转型者（success）与失败回弹者（snapback）的两类轨迹，分析评分、票房与导演异质性等协变量对转型结果的影响。

## 3. 数据说明

### 3.1 数据来源与处理流程

项目数据主要来自 IMDb 公开数据集（https://datasets.imdbws.com/ ）。我们在后端建立了一套离线流水线来完成数据清洗和统计计算，前端只负责消费预计算好的数据契约，不在运行时进行任何重算。

### 3.2 当前数据契约（`public/data/*.json`）

目前前端加载的数据契约包括六份 JSON 文件：

- `genres.json` 存储了 21 个电影类型。
- `actors.json` 包含演员的早期画像、聚类结果、投影坐标以及转型标注。
- `films.json` 是按演员展开的影片序列，涵盖评分、投票数和主类型。
- `entropy.json` 记录了每位演员的类型香农熵曲线。
- `markov.json` 是按群落与阶段预计算的类型转移矩阵。
- `alignment.json` 保存了 T=0 对齐后的轨迹与协变量数据。

### 3.3 当前数据规模

目前数据覆盖 814 位演员，共计 20,811 条影片记录，涉及 21 个类型。演员被聚为 8 个群落，每个群落规模约在 79 到 120 人之间。对齐轨迹共有 707 条，其中成功转型 626 条、回弹 41 条、无转型 40 条。马尔可夫转移矩阵预计算了 24 个，对应 8 个群落分别乘以 early、mid、late 三个阶段。

## 4. 设计方案

系统采用“宏观 → 中观 → 微观”的四视图协同框架。

**视图 A：Genre-Space Cluster（散点投影）** 将每位演员映射为一个散点，坐标来自早期类型向量的降维结果，使用颜色编码演员的主导类型。这个视图主要用于群落识别和分群选择。

**视图 B：Career River（时序河流）** 以作品序列索引为横轴（而不是自然年份），在横轴的影片节点上用垂直位置编码 IMDb 评分，节点大小映射投票数（反映影片热度）。河流图中的流带表达类型占比的演化过程，同时在河流图上叠加熵曲线以直观展现类型多样性的变化。

**视图 C：Transformation Alignment（对齐分叉）** 以首次转型为 T=0，对不同演员的类型熵曲线进行对齐，在同一张图内叠加选中演员组中每位演员的曲线，展示转型后成功与回弹两类轨迹的分叉形态，并支持按协变量进行过滤。

**视图 D：Markov Transition Gate（转移矩阵）** 以交互式热力矩阵呈现演员的类型转移概率，横轴为当前电影类型，纵轴为下一部电影类型，单元格颜色深浅表示转移概率大小。视图按 early、mid、late 三个阶段展示演员生涯不同时期的类型转移特征，并突出对角线以强化“转移”与“锁定”的视觉对比。

四个视图之间的核心联动逻辑如下：视图 A 的 brush 选择结果会驱动视图 B（展示群落平均态）和视图 D（展示对应群落的阶段矩阵）；点击视图 B 中的峰值节点可以触发视图 C 的个体高亮与同侪对齐；视图 C 则通过协变量过滤进行分层重组，为转型机制的解释提供支撑。

## 5. 目前实现进展

截至 2026-06-01，项目已经完成的任务如下：

1. 完成了数据清洗与预计算流水线的搭建，并成功在实际数据上运行。目前前端各个静态视图读取的都是真实数据。

2. 落地了应用骨架和数据脊柱。我们的前端工程基于 React + TypeScript + Zustand + D3 构建。`DataProvider` 实现了六份数据契约的一次性加载、形状校验与错误处理。索引结构（`actorsById`、`filmsByActor`、`markovByClusterStage`、`alignmentByActor`）均已建立，全局状态（brush、stage、selected actor/film、alignment filters）也已定义完毕。

3. 四个静态样例视图已经集成到主界面。视图 A、B、C、D 均可读取真实数据并渲染静态版本，其中视图 D 支持阶段切换（early/mid/late）。统一的面板容器（包含标题、工具栏、图例区、加载/错误态）也已实现完成。

## 6. 待完成的工作

接下来需要重点推进以下几个方面：

1. 三条联动链路的完整实现。这包括完善视图 A 到视图 B 和视图 D 的同步联动，实现视图 B 峰值点击到视图 C 高亮与详情面板的触发，以及稳定化视图 C 的协变量过滤重分层功能。

2. 从样例渲染升级为真正的分析态渲染。视图 B 需要完善单演员/群落双模式与空态策略；视图 C 则需要补齐同侪对齐、高亮策略与可读性优化（尤其是线条密度的控制）。

3. 通用交互组件与可用性的补全。这包括统一的 Tooltip、Legend、Slider、BrushLayer，以及提升联动可发现性（例如状态提示、当前 cohort 说明等）。

4. 提升数据与展示质量。我们会在现有设计基础上进一步优化清洗策略，重点处理类型之间数目不平衡、标签内部关联等问题，并在最终完成数据清洗后进一步优化视图呈现方式。

5. 部署与展示准备。我们需要完成 GitHub Pages 的自动化部署。

6. Case Study。我们计划使用此系统复现 1-2 个知名演员的“破圈”转型过程，作为项目的 Case Study 证明其在数据可视化分析方面的价值。
